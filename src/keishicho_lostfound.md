# keishicho_lostfound.py

[keishicho_lostfound.py](keishicho_lostfound.py)

警視庁 拾得物公表システム
(<https://www.ishitsu.keishicho.metro.tokyo.lg.jp/syutoku/>) の物品一覧を検索し、
CSV に書き出すコマンドラインツール。

分類・場所区分・地域はそれぞれカンマ区切りで複数指定でき、全組み合わせ
(直積) を順に検索して 1 つの CSV にまとめて出力する。サイト自体は 1 回の検索で
1 分類・1 場所・1 地域しか指定できないため、本ツールが組み合わせごとに
リクエストを繰り返す。

## 使い方

```bash
# カメラ類・上着類・ズボン類を、路上/建物と鉄道で、文京区・千代田区から、2026/05/25〜2026/05/28
python keishicho_lostfound.py \
    --category カメラ類,上着類,ズボン類 \
    --date_from 2026/05/25 --date_to 2026/05/28 \
    --place 路上・建物,鉄道 \
    --area 文京区,千代田区 \
    -o out.csv

# 拾得日付不明の物品を表示しない
python keishicho_lostfound.py --category 財布類 --date_from 2026/05/01 --date_to 2026/05/28 \
    --place 路上・建物 --area 新宿区 --no_include_unknown_date

# 指定できる分類・場所区分・地域名の一覧を表示
python keishicho_lostfound.py --list

# 実際にはリクエストせず、検索する組み合わせの URL だけ表示
python keishicho_lostfound.py -n --category 傘 --date_from 2026/05/01 --date_to 2026/05/28 \
    --place 鉄道 --area 港区
```

`--category` には大分類名 (例: `カメラ類`) と中分類名 (例: `上着類`, `ズボン類`) の
どちらも指定できる。大分類名を指定するとその大分類すべてが対象になる。
指定可能な名前は `--list` で確認できる。

過去 3 か月以内に届けられた落し物のみ検索可能 (サイト側の制約)。

## オプション

| オプション | 必須 | 既定値 | 説明 |
| --- | --- | --- | --- |
| `--category` | ○ | — | 分類名 (大分類または中分類)。カンマ区切りで複数指定可。 |
| `--date_from` | ○ | — | 拾得日 開始 (`YYYY/MM/DD`)。 |
| `--date_to` | ○ | — | 拾得日 終了 (`YYYY/MM/DD`)。 |
| `--place` | ○ | — | 場所区分 (`路上・建物`, `鉄道`, `バス`, `タクシー`, `その他交通機関`)。カンマ区切りで複数指定可。 |
| `--area` | ○ | — | 区市町村・都道府県名。カンマ区切りで複数指定可。 |
| `--include_unknown_date` | | True | 拾得日付不明の物品も表示する (既定)。 |
| `--no_include_unknown_date` | | — | 拾得日付不明の物品を表示しない。 |
| `-o`, `--output` | | 標準出力 | 出力 CSV パス。省略時は標準出力 (UTF-8)。ファイル出力時は UTF-8 (BOM 付き)。 |
| `--rows_per_page` | | 100 | 1 リクエストあたりの取得件数。 |
| `--delay` | | 0.5 | リクエスト間の待機秒数。 |
| `-n`, `--dry_run` | | — | リクエストせず、検索する組み合わせの URL を表示。 |
| `--list` | | — | 指定可能な分類・場所区分・地域名を一覧表示して終了。 |

## CSV の列

`分類`, `種類`, `特徴`, `在中品等`, `拾得日付`, `保管期間満了日`, `拾得場所`,
`拾得場所種別`, `問合せ先`, `問合せ電話番号`, `問合せ番号`, `検索場所区分`

`検索場所区分` は検索時に指定した場所区分 (`路上・建物` など) で、結果には
含まれないため補助列として付与している。

## 既知の問題

### 交通機関系の場所区分で同一物品が重複する (未修正)

`鉄道`・`バス`・`タクシー`・`その他交通機関` の遺失物は拾得場所が
「東京都以下不詳」となり、サイト側が住所コードを無視して全件を返す。
本ツールは場所区分 × 地域の全組み合わせを検索して結果をそのまま連結する
ため、これらの場所区分では同一物品が指定した地域数だけ重複する。

例: `--place 鉄道 --area 文京区,千代田区,中央区,港区,新宿区` だと、鉄道の
同じ物品が 5 件重複して出力される (5 区ぶん同じ結果が返るため)。
`路上・建物` は区単位で絞り込めるので重複しない。

回避するには、出力 CSV を `問合せ番号` 列で重複排除する。
